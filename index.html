How to use & limitations
  • Frontend: open index.html in a browser (double-click or Terminal: open index.html).
  • Backend + Gemini: in backend/, copy .env.example to .env, add GEMINI_API_KEY=your key (from Google AI Studio), optionally CLINIC_PUBLIC_PHONE= and CLINIC_OPERATIONS_EMAIL= so drafts use your real operations phone and email (not placeholders), run npm install once, then npm start.
  • Clinic knowledge: open the Clinic knowledge tab to manage saved Q/A. After generating, Transparency lists which knowledge ids the model cited.
  • Review library: each card can Suggest clinic knowledge — Gemini proposes Q/A from the case (answer grounded in your published reply); you edit in a dialog, then add to Clinic knowledge.
  • Review library (data): each row stores a single category (fixed list), past review, optional past internal context, and the final reply you published. The model uses the top similar rows to learn tone and handling—not “training” in the ML sense.
  • Generate: paste the review we received, optionally choose a category for that review (helps pick similar library examples), add optional what happened context for the AI (not posted as-is), then Generate new reply. Data POSTs to http://localhost:3000/generate-reply (no API key in the browser). If the server is off, you get the offline template instead.
